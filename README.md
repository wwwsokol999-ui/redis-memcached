## Задание 1. Кеширование

Кеширование нужно для того, чтобы не обращаться каждый раз к основной базе данных или к приложению за одними и теми же данными.

Какие проблемы оно решает:

- уменьшает количество обращений к БД;
- ускоряет получение часто запрашиваемых данных;
- снижает нагрузку на сервер;
- уменьшает задержки при работе приложения;
- позволяет не пересчитывать тяжелые данные заново;
- повышает общую производительность системы.

Пример:  
если пользователи часто открывают одну и ту же страницу или запрашивают один и тот же список товаров, то эти данные можно брать не из основной БД, а из кеша.

## Задание 3. Удаление по TTL в Memcached

Для проверки удаления ключей по TTL я записал в memcached несколько ключей со временем жизни 5 секунд.

Сначала выполнил команду:

```bash
printf "set key1 0 5 5\r\nvalue\r\nset key2 0 5 5\r\nhello\r\nget key1 key2\r\n" | nc localhost 11211
```
Сразу после записи сервер вернул значения ключей, значит они были успешно сохранены.

Далее через несколько секунд выполнил повторную проверку:

```bash
printf "get key1 key2\r\n" | nc localhost 11211
```
В ответ получил только END, а это значит, что ключи автоматически удалились после истечения TTL.

Таким образом я убедился, что механизм TTL в memcached работает корректно.



## Задание 4. Запись данных в Redis

Для выполнения задания я установил и запустил redis-server, после чего подключился к Redis через redis-cli.

Сначала проверил, что сервис Redis работает:

sudo systemctl status redis-server

После этого подключился к Redis и записал несколько ключей:

```bash
redis-cli
SET key1 value1
SET key2 value2
SET key3 value3
```

Далее посмотрел список ключей и их значения:

```bash
KEYS *
GET key1
GET key2
GET key3
```

В результате убедился, что ключи успешно записаны в Redis и доступны для чтения.

Также в ходе выполнения задания стало понятно, что Redis хранит данные в формате ключ -> значение, что позволяет очень быстро получать доступ к часто используемой информации.
